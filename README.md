# SE-TicTacToe

\# Tic-Tac-Toe（井字棋）- MVC C++项目



\##  项目简介



本项目是一个基于 \*\*C++ 控制台的井字棋游戏\*\*，采用经典的 \*\*MVC（Model-View-Controller）设计模式\*\*进行结构划分。



项目目标是练习：

\- 面向对象设计

\- 模块化开发

\- MVC架构思想

\- Git版本管理



\---



\##  项目结构设计（MVC）



本项目严格按照 MVC 分层设计：



\###  Model（模型层）

负责数据存储与游戏规则判断：

\- 棋盘数据管理（Board）

\- 落子逻辑

\- 胜负判断

\- 平局判断



\###  View（视图层）

负责所有用户交互：

\- 显示棋盘

\- 提示信息

\- 获取用户输入



\###  Controller（控制层）

负责游戏流程控制：

\- 控制玩家轮流

\- 调用 Model 处理逻辑

\- 调用 View 显示界面

\##  功能说明



\###  已实现功能

\- 3×3井字棋棋盘

\- 两人轮流下棋（X / O）

\- 合法性检测（防止非法落子）

\- 胜利判断（横 / 竖 / 对角线）

\- 平局判断

\- 输入范围提示（0-2）

\- 输入错误处理（防止崩溃）



\---



\##  MVC设计亮点



\###  解耦设计

\- Controller 不直接进行输入输出

\- View 不参与游戏逻辑判断

\- Model 独立处理数据和规则



\###  单一职责原则

每一层只负责自己的任务，提高代码可维护性
